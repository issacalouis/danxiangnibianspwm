# FULL 逆变控制固件用户使用手册

## 1. 适用范围

本手册适用于当前 `FULL` 工程固件，目标芯片为 STM32F407。固件实现单相逆变 SPWM 控制、OLED 状态显示、矩阵键盘参数设置、输入电流过流保护，以及 LED/蜂鸣器基础提示。

本手册按当前源码行为编写。若后续使用 STM32CubeMX 重新生成外设配置，必须重新核对 PWM、ADC、GPIO 和定时器配置是否仍与本手册一致。

## 2. 安全注意事项

该工程用于功率变换控制，调试时应按低风险流程上电：

1. 首次运行使用限流电源，并把输入电压调到低于额定值。
2. 不要在未确认 PWM 波形、死区、采样极性和保护阈值前连接满载。
3. 首次带功率级测试时，用示波器检查两桥臂 PWM 是否互补、是否存在异常直通风险。
4. 修改电压目标、电流限制、PI 参数或采样比例后，应重新从低压、轻载开始验证。
5. OLED 显示值来自 ADC 采样和软件换算，不能替代独立万用表、示波器或电流探头的安全判断。

## 3. 硬件接口摘要

### PWM 输出

| 功能 | 定时器通道 | 引脚 |
| --- | --- | --- |
| 桥臂 A 高边/主 PWM | TIM1_CH1 | PE9 |
| 桥臂 A 互补 PWM | TIM1_CH1N | PA7 |
| 桥臂 B 高边/主 PWM | TIM8_CH2 | PC7 |
| 桥臂 B 互补 PWM | TIM8_CH2N | PB0 |

PWM 周期当前为 `8399`，对应工程中的 SPWM 功率级控制输出。

### ADC 采样

| 测量量 | ADC 通道 | 引脚 | 软件换算 |
| --- | --- | --- | --- |
| 输出电压 | ADC1_CH5 | PA5 | `(Vadc - 1.72) * 55.8` |
| 输出电流 | ADC2_CH14 | PC4 | `(Vadc - 1.72) * 2.82` |
| 输入电流 | ADC2_CH15 | PC5 | `(Vadc - 1.72) * 2.82` |

`1.72V` 是当前软件使用的采样零点偏置。更换采样电路或传感器后，需要重新校准该偏置和比例系数。

### 显示与按键

| 设备 | 接口/引脚 |
| --- | --- |
| OLED | SSD1306 128x64，SPI3 |
| 4x4 矩阵键盘列线 | PD11、PE1、PE3、PE4 |
| 4x4 矩阵键盘行线 | PD3、PD4、PD7、PD10 |
| LED1 | PB8 |
| LED2 | PB2 |
| 蜂鸣器 | PE6 / TIM9_CH2 |

## 4. 上电流程

1. 确认功率输入、采样线、驱动电源、OLED 和键盘连接正确。
2. 使用 STM32CubeIDE 或调试器烧录固件。
3. 上电后 OLED 显示 `INITIALIZING`。
4. 初始化完成后进入 `INV CTRL` 主界面。
5. 默认输出电压目标为 `10.0V RMS`，默认输出电流限制为 `1.6A`。
6. TIM6 启动后，控制环按约 `20kHz` 运行，OLED 约每 `200ms` 刷新一次。

## 5. OLED 显示说明

主界面显示如下字段：

| 字段 | 含义 |
| --- | --- |
| `INV CTRL` | 逆变控制主界面 |
| `SET V` | 当前按键编辑对象为输出电压目标 |
| `SET I` | 当前按键编辑对象为输出电流限制 |
| `OC` | 已触发输入过流故障，PWM 回到中点输出 |
| `SV` | 输出电压目标，单位按 RMS 电压理解 |
| `LI` | 输出电流限制设定值 |
| `V` | 采样换算后的输出电压 |
| `Io` | 采样换算后的输出电流 |
| `Ii` | 采样换算后的输入电流 |
| `M` | 当前调制量，范围约为 `-0.96` 到 `0.96` |

## 6. 键盘操作

当前有效按键如下：

| 按键 | 功能 |
| --- | --- |
| `A` | 切换到电压目标编辑，OLED 右上角显示 `SET V` |
| `B` | 切换到电流限制编辑，OLED 右上角显示 `SET I` |
| `1` | 增加当前编辑项 |
| `2` | 减小当前编辑项 |
| `#` | 清除过流故障并复位 PI 积分 |

参数步进和范围：

| 参数 | 默认值 | 最小值 | 最大值 | 步进 |
| --- | ---: | ---: | ---: | ---: |
| 输出电压目标 `SV` | 10.0 | 0.0 | 60.0 | 1.0 |
| 输出电流限制 `LI` | 1.6 | 0.2 | 10.0 | 0.2 |

说明：当前固件没有把参数保存到 Flash。重新上电后会恢复源码中的默认值。

## 7. 保护与故障恢复

当前固件实现输入电流过流保护：

- 保护判断量：`Ii` 的原始换算值绝对值。
- 默认阈值：`2.5A`。
- 触发条件：`abs(i_in_raw) >= 2.5A`。
- 触发结果：`fault_oc` 置位，调制量清零，PWM 占空比回到 50% 中点，电压环和电流环 PI 积分清零，OLED 显示 `OC`。

恢复步骤：

1. 先降低输入电压或断开负载，确认过流原因已排除。
2. 按 `#` 清除故障。
3. 从低电压、轻载重新测试。

注意：`LI` 是输出电流环参考限幅，不等同于硬件级短路保护。实际功率级仍建议配置独立硬件保护，例如驱动器 DESAT、比较器关断或保险限流。

## 8. 推荐调试流程

### 空载低压测试

1. 不接负载，输入电源限流。
2. 设置较低输入电压。
3. 观察 OLED 是否正常刷新。
4. 用示波器检查 PE9/PA7、PC7/PB0 PWM 是否存在互补输出和死区。
5. 确认没有异常发热、异常蜂鸣或电源限流。

### 采样校准

1. 输出端无电压时，记录 OLED 的 `V` 是否接近 0。
2. 无电流时，记录 `Io`、`Ii` 是否接近 0。
3. 用独立仪表施加已知电压/电流，核对显示比例。
4. 若偏差明显，校准 `SAMPLE_BIAS`、`VOLTAGE_SAMPLE_RATIO`、`CURRENT_SAMPLE_RATIO`。

### 闭环测试

1. 设置 `SV = 10.0`，`LI = 1.6`。
2. 接轻载，缓慢提高输入电压。
3. 观察输出波形、`V`、`Io`、`Ii` 和 `M`。
4. 逐步调整目标电压和负载，不要一次跳到满功率条件。

## 9. 常见现象

| 现象 | 可能原因 | 处理建议 |
| --- | --- | --- |
| OLED 显示 `OC` | 输入电流超过阈值或采样偏置异常 | 降低输入/断开负载，检查采样，再按 `#` 清故障 |
| `V`、`Io`、`Ii` 空载不接近 0 | 采样零点偏置不准 | 校准 `SAMPLE_BIAS` 或检查模拟前端 |
| 调制量 `M` 长时间接近限幅 | 目标过高、输入电压不足、负载过重或 PI 参数不合适 | 降低 `SV`，减轻负载，检查母线电压和环路参数 |
| 按键无响应 | 键盘行列接线错误或扫描引脚不匹配 | 检查 PD3/PD4/PD7/PD10 与 PD11/PE1/PE3/PE4 |
| 输出波形异常 | PWM 同步、采样极性、驱动死区或桥臂接线问题 | 断开功率负载，用示波器逐项确认 |

## 10. 开发与构建

推荐使用 STM32CubeIDE 打开工程，必要时从 `FULL.ioc` 查看外设配置。

命令行构建：

```sh
cd Debug
make -j16 all
```

清理构建产物：

```sh
cd Debug
make -j16 clean
```

修改控制参数时，优先查看：

- `Core/Inc/boost.h`：目标范围、采样比例、PI 参数、保护阈值。
- `Core/Src/boost.c`：控制环、PWM 写入、ADC 回调、故障处理。
- `Core/Src/main.c`：初始化流程、OLED 刷新和按键扫描周期。
- `Core/Src/tim.c`：TIM1/TIM8 PWM、TIM6 控制周期、TIM9 LED/蜂鸣器 PWM。

## 11. 当前固件限制

1. 参数不会掉电保存。
2. 过流保护主要依赖输入电流软件判断，不是独立硬件快速关断。
3. OLED 显示刷新较慢，只适合观察趋势，不适合作为瞬态诊断依据。
4. 输出电压、电流的准确度取决于模拟采样电路和软件校准。
5. 若重新生成 CubeMX 代码，应重点检查 `USER CODE` 区域和 PWM/ADC/TIM 配置是否被覆盖。
