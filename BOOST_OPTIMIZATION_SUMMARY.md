# Boost电路PI控制优化总结

## 优化内容

根据反馈，对boost.c进行了以下4项优化：

---

## 1. ✅ 前馈控制 (Feedforward)

### 原理
Boost电路稳态时理想占空比: **D = 1 - Vin/Vout**

使用前馈可以让PI控制器只需补偿误差，而不是从零开始建立整个控制量，从而：
- 响应更快
- 超调更小
- 积分项从零附近工作

### 实现
```c
float duty_ff = 1.0f - v_in / v_target;
float duty_cmd = duty_ff + PI_correction;
```

### 配置
- 在 `boost.h` 中通过 `ENABLE_FEEDFORWARD` 宏开关控制（默认启用）
- 需要采集输入电压 `boost.v_in`
- 如果硬件不支持Vin采样，可设置 `ENABLE_FEEDFORWARD = 0` 禁用

### 硬件要求
需要配置ADC第二通道采样Vin，或者从电源模块获取Vin值。

---

## 2. ✅ 抗积分饱和 (Anti-windup)

### 问题
当占空比达到饱和限幅（90%或5%）时，如果误差方向还在让积分继续累积，会导致：
- 积分项过大
- 退出饱和后响应迟缓
- 超调增大

### 解决方案
检测饱和条件，冻结本次积分增量：

```c
if ((new_duty > BOOST_DUTY_MAX && error > 0.0f) ||
    (new_duty < BOOST_DUTY_MIN && error < 0.0f)) {
    /* 饱和且误差方向不利 → 回退Ki项 */
    new_duty -= boost.pi.ki * error;
}
```

### 效果
- 防止积分项过度累积
- 改善从饱和状态恢复时的响应
- 减少超调

---

## 3. ✅ ADC滤波

### 问题
单次ADC采样直接用于控制，开关纹波和噪声会进入控制环，导致：
- 控制量抖动
- 积分项受干扰
- 系统稳定性下降

### 解决方案
一阶低通滤波：

```c
boost.v_out = (1.0f - α) * boost.v_out + α * v_out_raw;
```

其中 α = 0.2 (可在boost.h中调整 `ADC_FILTER_ALPHA`)

### 效果
- 平滑ADC采样值
- 抑制高频噪声
- 提高控制稳定性

### 调整建议
- α越小，滤波越强，但响应越慢
- α越大，响应越快，但噪声抑制越弱
- 推荐范围: 0.1 ~ 0.3

---

## 4. ⚠️ ADC与PWM同步采样

### 问题
当前使用 `HAL_ADC_PollForConversion` 在1ms软件节拍中轮询ADC：
- 采样时刻与PWM相位随机漂移
- 可能采到开关瞬间的噪声尖峰
- 引入随机扰动

### 解决方案
使用TIM1触发ADC，在PWM周期的固定相位采样（例如开关管关断中点）

### 实现方式
详见 **ADC_PWM_SYNC_CONFIG.md** 文档，提供了两种方案：
1. **TIM1触发 + 中断模式** (推荐先实现)
2. **TIM1触发 + DMA模式** (最高效)

### 优势
- 采样时刻固定，避开开关噪声
- 降低CPU占用（中断/DMA模式）
- 提高采样一致性

### 注意
此项优化需要修改硬件配置（STM32CubeMX），不是纯软件修改。

---

## 参数调整建议

### PI参数
当前参数：
- Kp = 0.005
- Ki = 0.0002

启用前馈后，可以适当**减小Kp和Ki**，因为前馈已经提供了主要的控制量，PI只需补偿小误差。

建议尝试：
- Kp = 0.003 ~ 0.005
- Ki = 0.0001 ~ 0.0002

### 滤波系数
- 当前 α = 0.2
- 如果输出电压波动大，减小α（例如0.15）
- 如果响应太慢，增大α（例如0.25）

### 前馈开关
- 如果Vin稳定且已知，启用前馈效果最好
- 如果Vin波动大或无法准确采样，可以禁用前馈

## 测试建议

1. **阶跃响应测试**
   - 按键调整目标电压，观察上升时间、超调量、稳态误差
   - 对比优化前后的波形

2. **负载扰动测试**
   - 突加/突减负载，观察电压恢复时间和波动幅度

3. **输入电压扰动测试**
   - 改变输入电压，观察输出电压稳定性（前馈效果）

4. **示波器观察**
   - CH1: 输出电压Vout
   - CH2: PWM波形
   - CH3: ADC采样触发信号（如果实现同步采样）
